1. 중첩 클래스
    하나의 클래스 안에 다른 클래스를 선언하는 방식
    - 코드의 복잡성을 줄이기 위해 사용
    - 외부에서 내부 클래스를 숨길수 있음

    # 종류 (선언 위치에 따라 분류됨)
    1. 멤버 클래스 (클래스의 멤버로 선언) -> 자동차와 엔진의 관계?
       1. 인스턴스 멤버 클래스
       - 외부 클래스의 인스턴스 멤버처럼 동작하는 클래스
       - 외부 클래스의 멤버(필드,메서드)에도 쉽게 접근 가능
       - 외부클래스의 인스턴스를 먼저 생성해야 내부 클래스 사용 가능

       2. 정적 멤버 클래스
       - static 키워드가 붙은 내부 크래스
       - 외부 클래스의 인스턴스를 생성하지 않고도 사용 가능
       - 외부 클래스의 static 멤버만 접근 가능
         ex) 학교 공지 게시판(static)과 반 게시판(instance)의 개념?

    2. 로컬 클래스 -> 버튼 클릭 이벤트?
       - 메서드 내부에서만 선언, 해당 메서드가 실행될 때만 존재하는 클래스
       - 메서드 내부의 지역변수(상수)만 접근 가능하다

    3. 빌더 패턴
